# Actividad de la versión de Workfront Fusion: Semana del 15 de junio de 2026

Esta página describe todas las mejoras realizadas en Adobe Workfront Fusion durante la semana del 15 de junio de 2026.

Para obtener una lista de todos los cambios recientes, consulte [Actividad de la versión de Adobe Workfront Fusion](/help/workfront-fusion/fusion-product-releases/fusion-release-activity.md).

Para obtener una lista de las correcciones de errores recientes en Workfront Fusion, consulte la página [Actualizaciones de mantenimiento de Workfront](https://experienceleague.adobe.com/es/docs/workfront-known-issues/releases/current-updates) y busque cualquier actualización etiquetada como Actualización de mantenimiento de Workfront Fusion.

## Creación de paquetes de JavaScript personalizados para utilizarlos en escenarios

Para proporcionar una mejor flexibilidad y control de sus escenarios, hemos agregado la capacidad de crear paquetes de JavaScript personalizados que puede usar en escenarios. Puede crear paquetes personalizados en el área Paquetes de Fusion. A continuación, agregue funciones o variables de estos paquetes a los escenarios en forma de módulo App Builder de Adobe.

Los paquetes incluyen funciones, junto con cualquier variable o dependencia en la que se basen las funciones. También puede probar funciones en el paquete antes de usarlas en sus escenarios

Dado que los paquetes personalizados funcionan a través de Adobe App Builder, su organización debe tener una licencia de Adobe App Builder para utilizarlos.

Para obtener más información sobre el uso de funciones personalizadas en Fusion, consulte [Usar paquetes de funciones personalizadas](/help/workfront-fusion/create-scenarios/map-data/use-custom-function-packages.md).

## Ejecutar un módulo de bloque de código personalizado ya disponible

Para añadir flexibilidad a Workfront Fusion, hemos creado el módulo de bloque de código personalizado Adobe App Builder > Ejecutar. Ahora puede introducir un bloque de código personalizado directamente en el módulo, que se ejecutará cuando se ejecute el escenario.

Este nuevo módulo difiere del módulo App Builder > Ejecutar una función personalizada de Adobe en que se configura el bloque de código directamente en el módulo, en lugar de hacer referencia a una función del área Funciones.

Su organización debe tener Adobe App Builder para utilizar bloques de código personalizado y funciones personalizadas en Fusion.

Para obtener más información, consulte [Módulos App Builder de Adobe](/help/workfront-fusion/references/apps-and-modules/adobe-connectors/adobe-app-builder.md).


## Ver cambios entre versiones de escenarios

Para facilitar la comprensión de los cambios entre versiones de escenarios, se ha añadido la capacidad de ver y comparar dichos cambios. Ahora puede que aparezca una ventana que muestra cambios específicos entre dos versiones seleccionadas del mismo escenario.

Para obtener más información, vea [Ver y administrar versiones de escenarios](/help/workfront-fusion/manage-scenarios/restore-a-scenario-version.md).
